# SalesPilotAI CRM — Руководство по тестированию

## Требования

- Docker Desktop запущен
- Приложение поднято: `docker compose -f docker-compose.prod.yml up --build`
- Браузер открыт на `http://localhost`

---

## 1. Дашборд

**URL:** `http://localhost/`

1. Открыть страницу — должны отобразиться 6 KPI-карточек (Всего лидов, Конверсия, Всего сделок, Стоимость воронки, Прогноз выручки, Выигранные сделки).
2. При пустой БД все значения показывают `0` или `—` — это норма.
3. Нажать кнопку обновления (иконка справа от заголовка) — данные перезагружаются.
4. Два графика внизу: «Сделки по статусу» (bar chart) и «Лиды по статусу» (pie chart).

---

## 2. Лиды

**URL:** `http://localhost/leads`

### 2.1 Создание лида через API

Открыть `http://localhost/api/docs` — интерактивная документация Swagger.

Найти раздел **leads** → `POST /api/v1/leads` → «Try it out» → вставить JSON:

```json
{
  "first_name": "Иван",
  "last_name": "Петров",
  "email": "ivan@example.com",
  "phone": "+79001234567",
  "company": "ООО Ромашка",
  "source": "website",
  "owner_id": "00000000-0000-0000-0000-000000000001"
}
```

Нажать **Execute** — ответ `201` с созданным лидом.

### 2.2 Просмотр списка лидов

Перейти на `http://localhost/leads` — созданный лид появился в таблице.

### 2.3 Карточка лида

Кликнуть на строку лида — открывается страница с:
- Блок информации (имя, email, телефон, компания, источник, заметки)
- История активности (пустая при первом открытии)
- Правая колонка: ИИ-инсайты

### 2.4 ИИ-функции на карточке лида

> Требуется заполненный `OPENAI_API_KEY` в `.env`

- **ИИ-оценка лида** — нажать «Оценить» → AI анализирует лид и возвращает score 0–100% с рекомендациями.
- **Следующий шаг** — нажать «Предложить» → AI рекомендует конкретное действие с приоритетом.
- **ИИ генератор писем** — выбрать тон (Дружелюбный / Официальный / Настойчивый), нажать «Сгенерировать» → появляется готовое письмо с темой и текстом, кнопка «Копировать».

## 3. Воронка продаж (Kanban)

**URL:** `http://localhost/pipeline/00000000-0000-0000-0000-000000000001`

> Для отображения нужны Pipeline и Stage в БД.

### 3.1 Создание Pipeline через API

`POST /api/v1/pipelines` в Swagger:

```json
{
  "id": "00000000-0000-0000-0000-000000000001",
  "name": "Основная воронка",
  "owner_id": "00000000-0000-0000-0000-000000000001"
}
```

### 3.2 Создание этапов

`POST /api/v1/pipelines/{pipeline_id}/stages` для каждого этапа:

```json
{ "name": "Квалификация", "position": 1, "probability": 0.2 }
{ "name": "Предложение", "position": 2, "probability": 0.5 }
{ "name": "Переговоры", "position": 3, "probability": 0.75 }
{ "name": "Закрыто: победа", "position": 4, "probability": 1.0 }
```

### 3.3 Создание сделки

`POST /api/v1/deals`:

```json
{
  "title": "Сделка с ООО Ромашка",
  "owner_id": "00000000-0000-0000-0000-000000000001",
  "stage_id": "<id этапа из шага 3.2>",
  "pipeline_id": "00000000-0000-0000-0000-000000000001",
  "value_amount": "150000.00",
  "value_currency": "RUB",
  "contact_name": "Иван Петров"
}
```

### 3.4 Kanban-доска

- Открыть `http://localhost/pipeline/00000000-0000-0000-0000-000000000001`
- Карточки сделок отображаются в колонках по этапам
- **Drag & Drop** — перетащить карточку из одной колонки в другую — сделка перемещается, изменения сохраняются

---

## 4. Сделки

**URL:** `http://localhost/deals`

Таблица всех сделок. После создания сделки через API — она появится здесь со статусом, суммой, этапом и датами.

---

## 5. Аналитика (Дашборд после заполнения данных)

После создания лидов и сделок вернуться на `http://localhost/`:
- KPI-карточки показывают реальные числа
- Графики отображают распределение лидов и сделок по статусам
- Прогноз выручки рассчитывается с учётом вероятности закрытия по этапам

---

## 6. API-документация

- Swagger UI: `http://localhost/api/docs`
- ReDoc: `http://localhost/api/redoc`

Все эндпоинты доступны для ручного тестирования через интерфейс Swagger.

## Ожидаемое состояние при старте

| Страница | Ожидаемое |
|----------|-----------|
| `/` Дашборд | 6 карточек с `0` или `—`, пустые графики |
| `/leads` Лиды | Пустая таблица «Лиды не найдены.» |
| `/deals` Сделки | Пустая таблица «Сделки не найдены.» |
| `/pipeline/...` Воронка | Ошибка загрузки (pipeline не существует) — норма |
| `/api/docs` Swagger | Полный список эндпоинтов |
